## Goal

Replace deprecated `AVAssetImageGenerator.copyCGImage(at:actualTime:)` (deprecated macOS 15) with the async `AVAssetImageGenerator.image(at:)` API (macOS 13+) in `SelectedFilesGrid.swift`.

## Context

- File: `GPSMetadataEditor/Features/FileIntake/Views/SelectedFilesGrid.swift:204`
- Current: synchronous computed property `var videoPreviewImage: NSImage?` on `SelectedFile`
- Deprecation: `copyCGImage(at:actualTime:)` removed warning since macOS 15.0
- Replacement: `AVAssetImageGenerator.image(at:)` — async throwing, returns `(CGImage, CMTime)`, available macOS 13+
- Deployment target: macOS 26 — no backwards-compat needed

## Problem

`videoPreviewImage` is a synchronous computed property on a `struct SelectedFile`. Cannot directly call `async` from it. Must convert the call site or restructure.

## Approach

Convert `videoPreviewImage` from synchronous computed property to a free async function or move thumbnail loading into the view via `.task {}` modifier with `@State`.

### Preferred: async task in view, cached in @State

In `SelectedFilesGrid` (or its item subview), load thumbnail on `.onAppear` / `.task`:

```swift
@State private var thumbnail: NSImage? = nil

.task(id: file.url) {
    thumbnail = await loadThumbnail(for: file.url)
}
```

Free function (or static method):

```swift
func loadThumbnail(for url: URL) async -> NSImage? {
    let asset = AVURLAsset(url: url)
    let generator = AVAssetImageGenerator(asset: asset)
    generator.appliesPreferredTrackTransform = true
    guard let (cgImage, _) = try? await generator.image(at: .zero) else { return nil }
    return NSImage(cgImage: cgImage, size: .zero)
}
```

Remove `videoPreviewImage` computed property from `SelectedFile` entirely.

## Files

- `GPSMetadataEditor/Features/FileIntake/Views/SelectedFilesGrid.swift`

  - Add `@State private var thumbnail` to grid item view
  - Add `.task(id:)` modifier loading thumbnail asynchronously

## Success Criteria

1. Build produces zero deprecation warnings for `copyCGImage`.
2. Video thumbnails still appear in the file grid for video files.
3. No synchronous I/O on the main thread during grid render.
