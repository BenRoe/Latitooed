<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User confirmation to destructive write | The app must not write before the user explicitly confirms overwrite. |
| UI state to service boundary | Selected files and selected coordinate become metadata writer inputs. |
| Per-file results to UI | Diagnostics must be captured without exposing confusing raw command output as primary UI copy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Repudiation/Tampering | destructive confirmation | mitigate | Require blocking confirmation with direct overwrite/no-restore copy before any writer call. |
| T-03-03-02 | Tampering | batch ordering | mitigate | Use one awaited sequential loop over a snapshot of selected files. |
| T-03-03-03 | Denial of Service | duplicate batch starts | mitigate | Disable Apply Location while a batch is running. |
| T-03-03-04 | Information integrity | mixed media results | mitigate | Map videos to warnings and still-image writes to success/failure rows with compact summary. |
| T-03-03-SC | Scope creep | persistence/progress UI | mitigate | Keep Phase 4 features out of this plan. |
</threat_model>
