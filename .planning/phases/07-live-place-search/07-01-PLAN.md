    New top-level struct CoordinateSearchPanel:
    - Retain: @Bindable var viewModel: CoordinateSelectionViewModel
    - Add: @State private var debounceTask: Task<Void, Never>? (stores the pending debounce; cancelled on each new onChange)
    - Add: @State private var isDropdownVisible = false (view-local dropdown visibility)

    Body structure — a single VStack(alignment: .leading, spacing: AppDesign.Spacing.sm):

    Inner search row HStack(spacing: AppDesign.Spacing.sm):
      TextField("Search for a place", text: $viewModel.searchQuery)
        .textFieldStyle(.roundedBorder)
        Trailing overlay — X clear button:
          .overlay(alignment: .trailing) { if !viewModel.searchQuery.isEmpty { Button { clear action } label: { Image(systemName: "xmark.circle.fill").foregroundStyle(.secondary) }.buttonStyle(.plain).accessibilityLabel("Clear search field").padding(.trailing, AppDesign.Spacing.xs) } }
        Escape handler:
          .onExitCommand { viewModel.searchQuery = ""; debounceTask?.cancel(); viewModel.cancelSearch(); isDropdownVisible = false }

    X clear button action (same three steps as Escape minus onExitCommand): set viewModel.searchQuery = "", call debounceTask?.cancel(), call viewModel.cancelSearch(), set isDropdownVisible = false.

    REMOVE the Search Button("Search", ...) entirely (per D-01).

    onChange wired on the search row HStack (NOT on the TextField alone, so it fires after any programmatic query mutation):
      .onChange(of: viewModel.searchQuery) { _, newValue in
        debounceTask?.cancel()
        let trimmed = newValue.trimmingCharacters(in: .whitespacesAndNewlines)
        guard trimmed.count >= 3 else {
          viewModel.cancelSearch()
          viewModel.clearSearch()
          isDropdownVisible = false
          return
        }
        isDropdownVisible = true
        debounceTask = Task { @MainActor in
          try? await Task.sleep(for: .milliseconds(500))
          guard !Task.isCancelled else { return }
          viewModel.search()
        }
      }

    Sync isDropdownVisible with ViewModel collapseResults path (map tap, manual field entry, recent coordinate selection all call setCoordinate(collapseResults:true) which sets isSearchResultsExpanded = false):
      .onChange(of: viewModel.isSearchResultsExpanded) { _, newValue in
        if !newValue { isDropdownVisible = false }
      }

    Cleanup on disappear:
      .onDisappear { debounceTask?.cancel() }

    Floating dropdown overlay on the search row HStack:
      .overlay(alignment: .bottom) {
        if isDropdownVisible {
          dropdownContent
            .offset(y: /* positive value to push below the HStack — use 36 as baseline for standard TextField height, adjust if needed */)
        }
      }

    Private computed property dropdownContent — a VStack(alignment: .leading, spacing: AppDesign.Spacing.sm):
      - "Results" label: Text("Results").font(.caption).foregroundStyle(.secondary)
      - Spinner (when searching): if viewModel.searchStatus == .searching { ProgressView().controlSize(.small).accessibilityLabel("Searching") }
      - Status text (non-idle, non-searching): if let statusText = viewModel.searchStatusText { Text(statusText).font(.caption).foregroundStyle(viewModel.searchStatus == .failed ? .orange : .secondary) }
      - Results list: ScrollView { VStack(alignment: .leading, spacing: 0) { ForEach(viewModel.searchResults) { result in Button { viewModel.selectSearchResult(result); isDropdownVisible = false } label: { CoordinateSearchResultRow(result: result) }.buttonStyle(.plain) } } }.frame(maxHeight: 240)
      Container styling on the outer VStack:
        .frame(maxWidth: .infinity)
        .padding(AppDesign.Spacing.sm)
        .background(.regularMaterial)
        .clipShape(.rect(cornerSize: AppDesign.Radius.mediumSize))
        .shadow(color: .black.opacity(0.12), radius: 8, x: 0, y: 4)
        .accessibilityElement(children: .contain)

    The inline if viewModel.isSearchResultsExpanded { ... } expanding block from the original file is REMOVED entirely (replaced by the overlay).

    IMPORTANT — overlay offset: .overlay(alignment: .bottom) anchors the overlay's bottom edge to the HStack's bottom edge. To position the dropdown BELOW the field, apply .offset(y: dropdownHeight) is not needed — instead use alignment: .bottom with the overlay content having its own padding so it appears below. The research notes that a positive Y offset pushes the overlay downward. Test with offset(y: 36) first (standard roundedBorder TextField height on macOS). If the dropdown renders on top of the field instead of below it, switch to a ZStack approach with explicit padding.

    IMPORTANT — Swift 6 / strict concurrency: the debounce Task is annotated Task { @MainActor in ... } so all viewModel accesses inside it are on the main actor. This is required because CoordinateSelectionViewModel is @MainActor @Observable.

    IMPORTANT — no Combine import. Use Task.sleep only.
