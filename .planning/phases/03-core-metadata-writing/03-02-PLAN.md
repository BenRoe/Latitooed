<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App bundle to executable helper | The helper artifact must be the bundled ExifTool, not a system binary. |
| App process to child process | Child process receives user-selected file paths and coordinate arguments. |
| Security-scoped URL to helper write | File access must remain valid while ExifTool writes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | helper resolution | mitigate | Resolve only from `Bundle.main`; test no Homebrew/PATH fallback strings exist. |
| T-03-02-02 | Elevation | subprocess invocation | mitigate | Use `Process.executableURL` and `arguments`, never a shell. |
| T-03-02-03 | Tampering | file write access | mitigate | Hold security-scoped access for the full runner call. |
| T-03-02-04 | Denial of Service | active child process | mitigate | Terminate active process when the surrounding Swift task is cancelled. |
| T-03-02-SC | Supply chain | bundled helper | mitigate | Add helper from official ExifTool distribution and preserve provenance/licensing notes for Phase 5 hardening. |
</threat_model>
