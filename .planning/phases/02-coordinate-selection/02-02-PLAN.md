<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User query to MapKit service | Search text is user-controlled and leaves app process through Apple MapKit APIs. |
| MapKit result to selected coordinate | External search data becomes app target coordinate state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Information Disclosure | MapKit search | mitigate | Only send explicit user-entered queries on Search/Return; do not live-search every keystroke. |
| T-02-02-02 | Spoofing | search result selection | mitigate | Store/display lightweight title/subtitle and coordinate only; ready status confirms final coordinate. |
| T-02-02-03 | Denial of Service | repeated searches | mitigate | Cancel previous active search before starting replacement and ignore stale responses. |
| T-02-02-04 | Repudiation | search failure states | mitigate | Surface quiet inline failure copy while treating cancellation as normal lifecycle. |
| T-02-02-SC | Tampering | third-party APIs | accept | Apple MapKit only; no Google, no third-party SDKs, no new packages. |
</threat_model>
