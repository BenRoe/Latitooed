# Phase 09.1 Plan 01: Swift Concurrency and Test Quality Fixes Summary

Eliminated a Task leak in FileIntakeViewModel via a stored @ObservationIgnored handle with cancel-before-assign and deinit cancellation; fixed five test quality findings including sleep-based polling, overly broad error-type assertions, for-loop parameterized tests, in-helper #expect calls, and a missing @unchecked Sendable comment.

## Decisions Made

- `@ObservationIgnored` on `activeGPSScanTask` prevents the `Task<Void, Never>?` handle from triggering `@Observable` view updates on assignment.
- `deinit` cancellation is safe: `Task.cancel()` is synchronous and only sets a cooperative cancellation flag; no `await` in deinit.
- `RecordingGPSReadRunner` converted from `struct` to `final class` so `capturedArguments` can be mutated inside `func run(...)` without requiring `mutating func` (which `ProcessRunning` protocol does not permit).
- `#_sourceLocation` replaces the 4-parameter `fileID/filePath/line/column` form for cleaner call sites.
