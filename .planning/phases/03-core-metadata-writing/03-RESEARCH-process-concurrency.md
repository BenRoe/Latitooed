### Planning implications

- Define `MetadataWriter: Sendable` with a single async write method, for example `func writeGPS(to file: SelectedMediaFile, coordinate: CoordinateSelection) async -> MetadataWriteResult`. Returning a non-throwing structured result is preferable for per-file failures because META-07 requires user-facing status plus diagnostics, and a batch should continue after one file fails. [VERIFIED: `.planning/REQUIREMENTS.md`] [VERIFIED: `.planning/phases/03-core-metadata-writing/03-CONTEXT.md`]
- Keep a separate `BatchMetadataWriteService` or view-model method that loops over current `SelectedMediaFile` snapshots in order. It should call `try Task.checkCancellation()` before each file even though Phase 3 has no cancellation UI, so Phase 4 can add cancellation without changing the writer contract. [VERIFIED: `.planning/ROADMAP.md`] [VERIFIED: `.agents/skills/swift-concurrency-pro/references/cancellation.md`]
- Filter by `MediaFileKind`: `.jpeg` and `.heic` call the writer; `.mov` and `.mp4` return warning results because video writing is deferred to Phase 4. The writer should still return structured failures for missing, inaccessible, or unwritable still files at write time. [VERIFIED: `.planning/phases/03-core-metadata-writing/03-CONTEXT.md`] [VERIFIED: `GPSMetadataEditor/Features/FileIntake/Models/MediaFileKind.swift`]
- Put argument construction in `ExifToolArgumentBuilder`, not inside process-launch code. The builder should output an ordered `[String]` containing destructive overwrite behavior, GPS position, and the file path as its own final argument; this makes D-08 testable without launching ExifTool. [VERIFIED: `.planning/phases/03-core-metadata-writing/03-CONTEXT.md`] [VERIFIED: Context7 `/exiftool/exiftool`]
- Recommended still-image arguments: `["-overwrite_original", "-gpsposition=\(coordinate.latitude), \(coordinate.longitude)", fileURL.path(percentEncoded: false)]`. The `-overwrite_original` flag follows locked D-01; the decimal `GPSPosition` strategy follows ExifTool FAQ examples. [VERIFIED: `.planning/phases/03-core-metadata-writing/03-CONTEXT.md`] [VERIFIED: Context7 `/exiftool/exiftool`] [ASSUMED: exact HEIC behavior still needs host sample verification]
- Shape the process seam as `protocol ProcessRunning: Sendable { func run(executableURL: URL, arguments: [String]) async throws -> ProcessResult }`, with `ProcessResult` containing `exitStatus: Int32`, `stdout: String`, and `stderr: String`. Keep `Process` itself local to the concrete runner because it is mutable reference state and is not a good Sendable dependency surface. [CITED: https://developer.apple.com/documentation/foundation/process] [VERIFIED: `.agents/skills/swift-concurrency-pro/references/bridging.md`] [ASSUMED: implementation will satisfy strict-concurrency diagnostics without additional actor wrapping]
- Implement the runner with one checked continuation that resumes exactly once from `terminationHandler`; the continuation path must also handle failure before launch. The local concurrency reference is explicit that checked continuations must resume exactly once on every path. [VERIFIED: `.agents/skills/swift-concurrency-pro/references/bridging.md`] [CITED: https://developer.apple.com/documentation/foundation/process/1408746-terminationhandler]
- Avoid `waitUntilExit()` in app async code because Apple documents it as blocking the current process until the subprocess finishes; use a termination-handler bridge instead. [CITED: https://developer.apple.com/documentation/foundation/process/1415808-waituntilexit] [VERIFIED: `.agents/skills/swift-concurrency-pro/references/bridging.md`]
- Read stdout/stderr after process termination for Phase 3 simplicity. If later output volume becomes large enough to risk pipe back-pressure, Phase 4/5 can switch to streaming output, but ExifTool single-file GPS writes should be small. [CITED: https://developer.apple.com/documentation/foundation/pipe] [ASSUMED]
- The cancellation handler's `onCancel` may run on any thread according to the local concurrency reference, so it should only call thread-safe process cancellation mechanics or hop through a small actor/lock-protected cancellation box if strict concurrency requires it. Do not mutate `@MainActor` UI state from `onCancel`. [VERIFIED: `.agents/skills/swift-concurrency-pro/references/cancellation.md`]
- Resolve the helper from `Bundle.main` in a dedicated resolver and fail structurally if missing or not executable, but keep final packaging/signing hardening for Phase 5. [VERIFIED: `.agents/skills/swift-concurrency-pro/references/bridging.md`] [ASSUMED]
