<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local file names to card UI | Filenames and folders may be long or user-controlled strings. |
| Visual status to user decisions | Users rely on card status when deciding whether to apply metadata or inspect results. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Spoofing | status badges | mitigate | Include text plus SF Symbol for GPS/result states; do not rely on color only. |
| T-06-02-02 | Information Disclosure | filename rendering | mitigate | Truncate long filenames in cards and expose full names through `.help`, without adding new persisted storage. |
| T-06-02-03 | Denial of Service | large loaded batches | mitigate | Use `LazyVGrid` inside `ScrollView` and avoid eager thumbnail generation or per-card unstructured tasks. |
| T-06-02-SC | Supply chain | UI dependencies | accept | Use only SwiftUI and SF Symbols; no third-party UI or image-loading framework. |
</threat_model>
