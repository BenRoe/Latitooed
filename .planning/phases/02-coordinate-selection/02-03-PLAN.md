<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Map interaction to coordinate state | User clicks on MapKit content to create coordinate state used later for metadata writes. |
| MapKit rendering/search UI | Apple map/search data appears inside the app UI. |
| Root file-intake shell to coordinate feature | New right-panel feature integrates with existing file-intake workflow. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Spoofing | target marker | mitigate | Render a pin only for real selected coordinates and reflect target state in the ready status row. |
| T-02-03-02 | Tampering | map click conversion | mitigate | Convert click point through MapKit APIs and run resulting coordinate through the same selected-coordinate state path. |
| T-02-03-03 | Information Disclosure | map/search UI | mitigate | Do not request current user location in Phase 2; default to Berlin and search only on explicit user action. |
| T-02-03-04 | Denial of Service | UI layout | mitigate | Keep dense controls above map and collapse stale result lists so map remains usable in the right panel. |
| T-02-03-SC | Tampering | UI dependencies | accept | Native SwiftUI and MapKit only; no third-party UI frameworks or registries. |
</threat_model>
