# Phase 1 - UI Design Contract

> Visual and interaction contract for the native macOS SwiftUI app shell and file intake workflow. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual native SwiftUI design system |
| Preset | Not applicable |
| Component library | Native SwiftUI controls only |
| Icon library | SF Symbols |
| Font | System font through SwiftUI text styles |

Implementation must use SwiftUI system controls and adaptive platform styling. Do not introduce third-party UI frameworks, custom web-style component registries, UIKit/AppKit color bridging in SwiftUI views, or fixed font sizes in implementation. Design token sizes below define hierarchy references for the planner; implementation should express them through SwiftUI text styles and shared design constants where possible.

---

## Layout Contract

| Region | Contract |
|--------|----------|
| Main window | Single utility-style macOS window with a split layout. Left side owns file intake and review. Right side is reserved for future coordinate/map functionality. |
| Launch state | Empty state is drop-zone-first. The drop zone is the primary visual anchor before files are selected. |
| Populated state | Replace the large drop zone with a compact persistent drop strip above the selected-files table. |
| Left column | Top add/drop controls, middle selected-files table, bottom details and latest warning area. |
| Right side | Quiet placeholder only. It must not show map controls, coordinate fields, batch controls, or any affordance that implies Phase 2 or later features exist. |
| Footer/status area | Small, low-emphasis area reserved for later location/batch state. In Phase 1 it may show neutral file-count or readiness text only. |

---

## Visual Hierarchy

| Priority | Element | Rule |
|----------|---------|------|
| 1 | Drop zone or compact drop strip | Primary focal point for the intake workflow. Use the accent only when actionable or drag-targeted. |
| 2 | Selected-files table | Main working surface after files are added. Rows must scan by display name first, then type, GPS status, and latest result. |
| 3 | Latest warning/details area | Persistent support surface for the most recent intake event. Use warning iconography and semantic warning color, not accent color. |
| 4 | Right-side placeholder | Visually quiet secondary surface with no competing focal treatment. |
| 5 | Footer/status | Lowest-emphasis summary text. |

Every icon-only visual affordance must be backed by a text label for VoiceOver and Voice Control. Prefer `Button("Label", systemImage: ..., action: ...)`, `Label`, `ContentUnavailableView`, `Table`, `List`, and native SwiftUI table/list selection behavior.

## Spacing Scale

Declared values are multiples of 4 and use the standard GSD scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal gaps, table cell micro-spacing |
| sm | 8px | Compact control spacing, row internal spacing, warning item gaps |
| md | 16px | Default control groups, table-to-detail spacing, drop-strip padding |
| lg | 24px | Column padding, empty-state grouping, bottom detail region padding |
| xl | 32px | Split-region internal gap, large drop-zone content grouping |
| 2xl | 48px | Empty-state vertical breathing room |
| 3xl | 64px | Maximum page-level empty-state spacing only |

Exceptions: macOS control intrinsic sizes and platform table row heights may use system metrics. Interactive controls must still maintain at least a 44px activation area where the visual target could otherwise become too small.

---

## Typography

Use exactly four hierarchy sizes and exactly two weights. Implementation should prefer SwiftUI text styles rather than hard-coded font sizes.

| Role | Size | Weight | Line Height | SwiftUI Style |
|------|------|--------|-------------|---------------|
| Label | 12px | Regular 400 | 1.4 | `.caption` or equivalent platform label style |
| Body | 14px | Regular 400 | 1.5 | `.body` |
| Heading | 17px | Bold 700 | 1.25 | `.headline` with `bold()` only when needed |
| Display | 24px | Bold 700 | 1.2 | `.title2` for the launch empty state only |

Do not use `.caption2`. Do not add extra weights such as medium or semibold during Phase 1. Long filenames must truncate in table rows with accessible full text available through row detail, tooltip, or the bottom detail area.

---

## Color

Use adaptive native colors. Hex values are light-mode design references, not fixed implementation colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `appBackground` reference `#F5F5F5` | Window background and broad empty space |
| Secondary (30%) | `contentSurface` reference `#FFFFFF` | Left column table surface, right placeholder surface, bottom detail surface |
| Accent (10%) | System accent color, blue reference `#0A84FF` | Primary CTA, drag-over border, selected row focus, keyboard focus ring |
| Warning | `warning` reference `#B26A00` | Unsupported, inaccessible, read-only, locked, missing, duplicate, or directory rejection states |
| Destructive | `destructive` reference `#D70015` | Future destructive overwrite actions only; no destructive action exists in Phase 1 |

Accent reserved for: Add Files primary CTA, active drop-target border, selected row focus, keyboard focus ring. Accent must not be used for all buttons, all links, row status badges, warnings, or the right-side placeholder.

## Component Contract

| Component | Contract |
|-----------|----------|
| Empty intake state | Use a large native drop zone with an add-files button and concise empty-state copy. Prefer `ContentUnavailableView` when it fits the final structure. |
| Compact drop strip | Persistent strip above the table after files exist. It accepts drops and contains the primary add-files button. |
| Add files action | Use a labeled button with SF Symbol `plus` or `plus.circle`. It opens the multi-select file picker. |
| Selected-files table | Columns: Display Name, Type, GPS, Latest Result. The table must support long filenames without layout breakage. |
| Type badge | Compact textual badge: JPEG, HEIC, MOV, MP4. Do not color-code type as the only differentiator. |
| GPS status | Phase 1 default is "Not checked" with a neutral icon. Future states may include "No GPS", "Has GPS", and "Updated", but Phase 1 must not claim metadata was read. |
| Latest result | Phase 1 default is "Pending". Rejected files do not enter the table. Accepted files can show neutral intake status only. |
| Row detail area | Shows selected filename and containing folder, plus access/warning detail. It should not duplicate the entire full path in every row. |
| Warning summary | Shows latest intake event warnings and lists every rejected item for bulk drops. It persists until the next picker/drop event. |
| Right placeholder | Plain native surface with quiet copy that location selection is planned later. No map imagery or disabled coordinate controls. |

---

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| File picker | Primary CTA opens a multi-select picker limited to JPEG, HEIC, MOV, and MP4 where platform type filters are available. |
| Drag and drop | Empty drop zone and compact drop strip both accept local file drops. Drag-over state uses accent border plus non-color state change. |
| Duplicate URL | Re-selecting or re-dropping the same URL must not create a new row. Surface the duplicate in the latest warning details if useful. |
| Unsupported file | Reject before table insertion and include the item in the latest warning details. |
| Directory drop | Reject in Phase 1 with clear copy; do not recursively scan. |
| Table selection | Selecting a row updates the bottom detail area. Do not use the right placeholder for file details. |
| Keyboard | Add-files action, table rows, and warning details must be reachable by keyboard. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Add Files |
| Empty state heading | Drop media files here |
| Empty state body | Add JPEG, HEIC, MOV, or MP4 files to review them before choosing a location. |
| Compact drop strip | Drop more files here or add files from Finder. |
| Right placeholder heading | Location selection comes next |
| Right placeholder body | Phase 1 is focused on building a reliable file set. |
| GPS placeholder | Not checked |
| Latest result placeholder | Pending |
| Unsupported file warning | "{filename}" is not a supported media file. Add JPEG, HEIC, MOV, or MP4 files. |
| Directory warning | "{filename}" is a folder. Add individual media files for now. |
| Missing file warning | "{filename}" could not be found. Choose the file again from Finder. |
| Inaccessible file warning | "{filename}" could not be accessed. Check permissions or choose another copy. |
| Read-only file warning | "{filename}" is read-only. Choose a writable copy before editing metadata. |
| Locked file warning | "{filename}" is locked. Unlock it in Finder before adding it. |
| Destructive confirmation | None in Phase 1. Metadata writes and overwrite behavior are out of scope. |

Avoid generic dialog labels such as Submit, OK, Cancel, or Save for Phase 1-specific actions. If a dismissal action is needed in a warning surface, use "Dismiss Warning".

## Accessibility Contract

| Area | Requirement |
|------|-------------|
| Text | Use Dynamic Type-compatible SwiftUI text styles. Do not force fixed font sizes in implementation. |
| Buttons | All buttons use text labels, including image-backed labels. Visual icon-only presentation may use `.labelStyle(.iconOnly)` only when the text label remains present for accessibility. |
| Color | Warnings and drag states must include icon, stroke, copy, or shape changes so color is not the only signal. |
| Motion | Drag-over and notice transitions must be subtle and respect Reduce Motion. |
| File names | Long, Unicode, and space-containing filenames must remain readable, selectable, and available to assistive technologies. |
| Table | Column headers must be clear and row selection must announce the selected file context. |
