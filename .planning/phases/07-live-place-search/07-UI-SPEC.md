## Spacing Scale

Reuse existing `AppDesign.Spacing` tokens unchanged. No new values are introduced.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4 pt | Icon-to-text gap inside result rows; clear-button inset |
| sm | 8 pt | Padding inside dropdown container; row inner padding; gap between search field and X button |
| md | 16 pt | Vertical gap between search row and fields below it in `CoordinateSelectionView` |
| lg | 24 pt | Outer padding of `CoordinateSelectionView` (unchanged, not a new token) |
| xl | 32 pt | Not used in new dropdown surfaces |

Reuse existing SwiftUI semantic font sizes. No new font sizes or weights are introduced.

| Role | SwiftUI Font | Weight | Usage |
|------|-------------|--------|-------|
| Result title | `.body` | regular (400) | Place name in each dropdown row — matches `CoordinateSearchResultRow` |
| Result subtitle | `.caption` | regular (400) | Address / administrative region below title — matches `CoordinateSearchResultRow` |

Line heights: SwiftUI system defaults (approximately 1.3 for `.body`, 1.2 for `.caption`). Do not override — system Dynamic Type handles this.

## Color

All colors derive from SwiftUI semantic and material tokens. No hex values are hard-coded — this is a macOS adaptive-appearance app that must honour light/dark and vibrancy automatically.

| Role | SwiftUI Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `.background` | `CoordinateSelectionView` outer background (unchanged) |
| Secondary (30%) | `.regularMaterial` | Dropdown container background — matches existing results panel |
| Accent (10%) | `.tint` | `mappin.circle` SF Symbol in result rows — matches existing `CoordinateSearchResultRow` |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for: the `mappin.circle` icon tint inside each result row. Do not apply tint to the X clear button, the search field border, the status text, or any other element in this phase.

Dropdown shadow: `.shadow(color: .black.opacity(0.12), radius: 8, x: 0, y: 4)` — gives visual lift over the map without creating a harsh hard shadow. Use only on the dropdown container, not on individual rows.

## Interaction Contract

### Search trigger

- `onChange(of: viewModel.searchQuery)` fires on every character change.
- If `query.count < 3` after trimming whitespace: cancel in-flight task, clear `searchResults`, set `isDropdownVisible = false` — no network call (D-05).
- If `query.count >= 3`: start a `Task` that calls `Task.sleep(for: .milliseconds(500))` then checks `Task.isCancelled` before calling `viewModel.search()` — this is the debounce (D-06). Cancel the previous `Task` when a new character arrives (D-07).

### Dropdown visibility rules

| Condition | Dropdown state |
|-----------|---------------|
| `query.count < 3` | Hidden (dismissed silently) |
| `query.count >= 3` AND search in-flight | Visible — shows spinner + "Searching..." |
| Results returned, count > 0 | Visible — shows result rows |
| Results returned, count == 0 | Visible — shows "No places found" copy |
| Search failed | Visible — shows error copy |
| User taps a result | Hidden (D-10) |
| User taps X button | Hidden |
| Escape key pressed | Hidden |
| User clicks outside dropdown | Hidden (use `.onTapGesture` on a transparent full-screen backdrop or `focusedValue` dismissal — executor chooses macOS-idiomatic approach) |

### Dropdown layout

- Width: matches the `TextField` width exactly. Achieved by applying `.overlay(alignment: .bottom)` on the `HStack` that contains the `TextField`, offset so the dropdown top edge aligns with the field bottom edge.
- Max height: 240 pt (approximately 5 result rows). If results exceed this, the list scrolls inside a `ScrollView`. Use `ScrollView { VStack(...) }` not `List` (avoids macOS List selection chrome).
- Z-order: the overlay sits above `CoordinateFieldsView`, `RecentCoordinatesView`, and the map. This is automatic with `.overlay` — it does not push layout down (D-04).
- Corner radius: `AppDesign.Radius.mediumSize` (8 pt) on all four corners of the container.

### Keyboard / focus

- `.onExitCommand` on the `TextField` handles Escape (D-09): clears query, cancels task, hides dropdown.
- Tab/arrow key navigation inside the dropdown is out of scope for this phase. Standard macOS focus traversal applies.
- The `TextField` retains first-responder status while the dropdown is visible — do not steal focus.

### X clear button

- SF Symbol: `xmark.circle.fill`, font size inherits from context (do not set explicit size — let the system size it relative to the field height).
- Foreground style: `.secondary` (adaptive gray). Do not use `.tint` — accent is reserved for map pins only.
- Hit target: the button is inset by `AppDesign.Spacing.xs` (4 pt) from the field's trailing edge.
- Visible condition: `!viewModel.searchQuery.isEmpty` only. Hidden (`.opacity(0)` or conditional) when field is empty.
- Action: set `viewModel.searchQuery = ""`, call `viewModel.cancelSearch()`, set `isDropdownVisible = false`.
- Accessibility label: `"Clear search field"`.

### Result selection (D-10)

- Tap a row: call `viewModel.selectSearchResult(result)` — coordinate updates.
- `searchQuery` remains the typed text (do not clear it).
- Dropdown is dismissed immediately.
- No confirmation or animation delay.

## Accessibility Contract

macOS-specific requirements for this phase:

| Element | Requirement |
|---------|-------------|
| X clear button | `accessibilityLabel("Clear search field")`. Button role (`.buttonRole`) not set — default is correct. |
| Dropdown rows | Each `CoordinateSearchResultRow` button must have a synthesised label from `result.title` + `result.subtitle` joined by comma. Add `.accessibilityLabel("\(result.title)\(result.subtitle.map { ", \($0)" } ?? "")")` if VoiceOver does not concatenate automatically. |
| Dropdown container | `accessibilityElement(children: .contain)` — allows VoiceOver to enter and navigate rows. |
| Spinner | `accessibilityLabel("Searching")` to override the default spinner label. |
| Dropdown visibility | Use SwiftUI conditional rendering (not `.opacity(0)` / `.hidden()`) so VoiceOver does not read invisible elements. |

## Implementation Constraints (for executor)

1. `CoordinateSearchPanel.swift` is the only view file that changes.
2. `CoordinateSelectionViewModel.swift` changes are minimal: remove `performSearchOnSubmit()`, remove `isSearchButtonDisabled`, add a `clearSearch()` convenience if desired. The `search()` method and all generation/task-cancellation infrastructure remain.
3. The overlay must not alter `CoordinateSelectionView`'s `VStack` layout. The `CoordinateSearchPanel` VStack height must not grow when the dropdown is visible.
4. `.overlay(alignment: .bottom)` applied on the search row `HStack` (containing `TextField` + formerly `Button`) is the preferred approach. The overlay content should be wrapped in a `ZStack` with `.frame(maxWidth: .infinity)` to match field width.
5. Debounce: implement with `Task { try? await Task.sleep(for: .milliseconds(500)); /* then search */ }` stored in a local `@State var debounceTask: Task<Void, Never>?`. Cancel on each new `onChange`. Do NOT use Combine unless it is already imported in the file.
6. The dropdown `ScrollView` max height is 240 pt — enforced with `.frame(maxHeight: 240)` on the scroll view.
7. Swift 6 / strict concurrency: all access to `viewModel` properties inside `Task` closures must go through `@MainActor` — the ViewModel is already `@MainActor @Observable`, so `Task { @MainActor in ... }` or `await MainActor.run { ... }` is required if the Task is not itself `@MainActor`-isolated.
