<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Finder/file picker to intake service | User-selected local URLs can point to unsupported, missing, locked, read-only, or external-drive resources. |
| File system metadata to app state | Resource values are untrusted until classified and validated. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Tampering | `FileIntakeService` | mitigate | Reject unsupported, directory, inaccessible, read-only, locked, and missing resources before adding snapshots. |
| T-01-02-02 | Information Disclosure | file paths | mitigate | Store URLs for app logic but display only filename and containing folder in the detail area per D-17. |
| T-01-02-03 | Denial of Service | duplicate/bulk intake | mitigate | Deduplicate by URL and return bounded warning records for rejected entries. |
| T-01-02-SC | Tampering | package installs | accept | No package-manager installs are planned. |
</threat_model>
