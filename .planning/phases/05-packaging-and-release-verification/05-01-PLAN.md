<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repository fixtures to user files | Smoke fixtures are copied before mutation so tracked source files stay clean. |
| Built app bundle to helper process | The script verifies helper existence and executable bit inside the `.app`. |
| Host signing state to release claim | Codesign evidence supports signed-app verification but not public notarized distribution. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Information Disclosure | release fixtures | mitigate | Use tiny nonsensitive fixtures and strip metadata; do not commit personal media with location data. |
| T-05-01-02 | Tampering | smoke files | mitigate | Copy fixtures to a temp directory before write smoke; never mutate tracked fixtures. |
| T-05-01-03 | Spoofing | helper verification | mitigate | Verify `Contents/Resources/ExifTool/exiftool` explicitly, not a PATH command. |
| T-05-01-04 | Repudiation | release evidence | mitigate | Record codesign and helper version commands in docs/UAT. |
</threat_model>
