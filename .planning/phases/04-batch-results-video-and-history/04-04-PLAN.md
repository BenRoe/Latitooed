<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Batch result state to persistence | In-memory per-file outcomes are summarized into durable counts. |
| Persisted history to active coordinate state | A previous coordinate value can be reused in the current batch flow. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Information Disclosure | batch history model/UI | mitigate | Persist and render counts only; never persist/display file paths, per-file results, diagnostics, or thumbnails. |
| T-04-04-02 | Tampering | history coordinate reuse | mitigate | Reuse only coordinate values through the existing coordinate validation/sync path. |
| T-04-04-03 | Reliability | post-write persistence failure | mitigate | Persistence errors must not rerun writes or corrupt existing selected-file results. |
| T-04-04-SC | Scope creep | restore/reopen behavior | mitigate | Do not add previous-file restore, reopen, or reconnect affordances. |
</threat_model>
